# 14.273 Mini Project — Igami (2017) Extension with Agglomeration

Extension of Igami (2017) "Making Oligopoly: Entry and Innovation in the Hard Disk Drive Industry" to study innovation spillovers / agglomeration effects in a dynamic oligopoly. Adds a `R = 3` regional dimension with bloc-pooled cost spillovers, a two-step structural estimator, three subsidy counterfactuals, an EU–US alliance counterfactual, and a misspecified-model strawman.

## Pipeline

```bash
cd code

# 1. Simulate firm-level data from the regional 2-period model
julia --project=. scripts/simulate_data.jl
# → data/simulated_data.csv  (500 markets, fixed seed)

# 2. Two-step estimator on the simulated data
julia --project=. scripts/estimate.jl
# → output/estimates/estimation.txt   (κ̂, φ̂, γ̂ macros)
# → output/tables/estimation.tex      (results table)

# 3. Counterfactuals — each consumes the estimates from step 2
julia --project=. scripts/run_innovation_subsidy.jl   # region-3 τ sweep
julia --project=. scripts/run_entry_subsidy.jl        # region-3 ψ sweep
julia --project=. scripts/run_joint_subsidy.jl        # joint (τ, ψ) grid + γ=0 control
julia --project=. scripts/run_merger.jl               # EU–US alliance (blocs = (1,1,2))
julia --project=. scripts/run_misspec.jl              # γ ≡ 0 strawman estimator

# 4. Re-render the joint-subsidy heatmaps from cached CSVs (no re-sweep)
julia --project=. scripts/plot_joint_subsidy.jl
# Reads output/cache/joint_subsidy_gamma_{hat,zero}.csv (~5 sec)
```

The writeup `writeup/progress_agglomeration.tex` `\input`s the macro and table files from `output/` directly:

```bash
cd writeup && latexmk -pdf progress_agglomeration.tex
```

## Model

**State** (per region, `R = 3`, each an `NTuple{3,Int}`): `(n_o, n_b, n_n, n_pe)` — old incumbents, "both"-tech incumbents, new entrants, potential entrants. Firms are pinned to a region for life.

**Demand (two-market, global):**

```
P_o = A − (B/M)·Q_o − (ρ·B/M)·Q_n
P_n = A − (B/M)·Q_n − (ρ·B/M)·Q_o
```

**"Both" firms** sell in both markets and internalize cannibalization — their new-gen output depresses their own old-gen price, deterring innovation.

**Bloc-pooled agglomeration:**

```
c_{n,r,t} = c_{n,0} + δ_r − γ_r · Σ_{r' ∈ bloc(r)} (n_{b,r',t} + n_{n,r',t})
```

`Params.blocs::NTuple{3,Int}` partitions the regions. Default `(1,2,3)` puts every region in its own bloc and recovers purely local spillovers; `(1,1,2)` represents an alliance of regions 1 and 2. `Params.c_n0_offset::NTuple{3,Float64}` defaults to `(0,0,0)` and is only nonzero in the misspec strawman, which sets `γ ≡ 0` and absorbs cross-region cost levels into δ_r.

**Solution.** One-pass backward induction. EVT1 private shocks make each firm's problem single-agent. Within each period the four stages (old → both → new → pe) act in order; within each stage the three regions move sequentially `r=1 → 2 → 3`. Each sub-stage is a scalar fixed point with closed-form logit CCPs.

## Key results

**Estimator** at `K = 500` markets (`output/estimates/estimation.txt`):

| Parameter | Truth | Estimate | s.e. |
|-----------|-------|----------|------|
| κ | 0.200 | 0.2193 | 0.0402 |
| φ | 0.200 | 0.1810 | 0.0403 |
| γ_r (all r) | 0.150 | 0.1500 | exact |

**Joint subsidy** (`output/estimates/joint_subsidy_estimates.txt`). Region-3 only; planner picks `(τ/κ̂, ψ/φ̂) = (0.10, 0.20)` with `ΔΣW = +0.01 %`. Under the `γ ≡ 0` control, the planner's argmax collapses to the corner `(0, 0)` — agglomeration is the externality the subsidy is correcting.

**EU–US alliance** at `K = 5000` MC markets with common random numbers (`output/estimates/merger_estimates.txt`):

| Region | ΔP(innov) | ΔP(enter) | ΔW |
|--------|-----------|-----------|-----|
| 1 (US)  | +4.71 % | +2.67 % | +6.46 % |
| 2 (EU)  | +6.14 % | +2.89 % | +6.59 % |
| 3 (RoW) | −3.89 % | −3.15 % | −0.07 % |
| Total   |         |         | +4.31 % |

The unallied region loses on innovation/entry but its share of the global CS gain almost exactly offsets its PS loss.

**Misspecified-model strawman** (`output/estimates/misspec_estimates.txt`). Re-estimating with `γ ≡ 0` and a region intercept `δ_r` on new-tech MC matches action-margin moments at the realized states by construction (innovator's-dilemma gap differs at the fourth decimal). Differences appear in markup heterogeneity (Lerner-new s.d. compressed from 0.071 to 0.058) and in the comparative-static response of innovation to a `+1 BOTH` shock to region 1 (`ΔP(innov)` is roughly three times more negative under misspec). See appendix of the writeup.
